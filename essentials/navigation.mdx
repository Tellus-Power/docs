The `navigation` field in `docs.json` controls the sidebar structure of your docs site. Every page you want to appear in the sidebar must be listed here — pages not included are hidden from navigation but remain accessible through search or direct links.

## Navigation syntax

The `navigation` field is an object. Use `tabs` for multi-section docs (e.g., separate Guides and API Reference sections) or `groups` for a single-section site.

<CodeGroup>

```json Single section (groups)
{
  "navigation": {
    "groups": [
      {
        "group": "Getting Started",
        "pages": ["quickstart", "development"]
      },
      {
        "group": "Guides",
        "pages": ["guides/authentication", "guides/webhooks"]
      }
    ]
  }
}
```

```json Multiple sections (tabs)
{
  "navigation": {
    "tabs": [
      {
        "tab": "Guides",
        "groups": [
          {
            "group": "Getting Started",
            "pages": ["quickstart", "development"]
          }
        ]
      },
      {
        "tab": "API Reference",
        "groups": [
          {
            "group": "Endpoints",
            "pages": ["api-reference/introduction", "api-reference/endpoint/get"]
          }
        ]
      }
    ]
  }
}
```

</CodeGroup>

## Folders

Put your MDX files in folders and reference them as `folder/filename` in `docs.json` (without the `.mdx` extension).

For example, a file at `guides/authentication.mdx` is referenced as `"guides/authentication"`.

```json docs.json
{
  "navigation": {
    "groups": [
      {
        "group": "Guides",
        "pages": [
          "guides/authentication",
          "guides/webhooks"
        ]
      }
    ]
  }
}
```

The corresponding URL is `https://yoursite.com/guides/authentication`.

<Warning>
  Do not use `api` as a top-level folder name. Mintlify reserves it internally. Use `api-reference` or another name instead.
</Warning>

## Nested groups

You can nest a group inside another group's `pages` array for deeper hierarchy:

```json docs.json
{
  "navigation": {
    "groups": [
      {
        "group": "Getting Started",
        "pages": [
          "quickstart",
          {
            "group": "Advanced Setup",
            "pages": ["advanced/custom-domain", "advanced/sso"]
          }
        ]
      }
    ]
  }
}
```

## Hidden pages

MDX files not listed in `navigation` are not shown in the sidebar. They are still:
- Findable through the search bar
- Accessible at their direct URL
- Linkable from other pages

This is useful for draft pages or content you want to share via direct link without surfacing in the sidebar.

## Updating navigation

Every time you add a new page, add its path to `docs.json`. The sidebar does not update automatically when you create new files.

<Tip>
  Run `mint dev` locally to see your navigation changes instantly before pushing to production.
</Tip>
