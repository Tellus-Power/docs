Every Mintlify site is configured through a `docs.json` file at the root of your documentation repository. Changes you make in this file take effect immediately in local preview and are deployed automatically when you push to your default branch.

## Identity

Set the name, theme, and visual identity of your site.

```json docs.json
{
  "name": "Acme Docs",
  "theme": "mint",
  "favicon": "/favicon.svg",
  "logo": {
    "light": "/logo/light.svg",
    "dark": "/logo/dark.svg",
    "href": "https://acme.com"
  }
}
```

<ResponseField name="name" type="string" required>
  Your project name. Appears in the browser tab title and site metadata.
</ResponseField>

<ResponseField name="theme" type="string">
  The visual theme. Options: `mint`, `maple`, `willow`, `sequoia`, `prism`, `luma`, `almond`, `aspen`, `palm`.
</ResponseField>

<ResponseField name="favicon" type="string">
  Path to your favicon, or a full URL. SVG is recommended for crisp rendering at all sizes.
</ResponseField>

<ResponseField name="logo" type="object">
  <Expandable title="Logo fields">
    <ResponseField name="light" type="string">Logo shown in light mode. Path or URL.</ResponseField>
    <ResponseField name="dark" type="string">Logo shown in dark mode. Path or URL.</ResponseField>
    <ResponseField name="href" type="string" default="/">URL the logo links to.</ResponseField>
  </Expandable>
</ResponseField>

## Colors

```json docs.json
{
  "colors": {
    "primary": "#0066FF",
    "light": "#66A3FF",
    "dark": "#004ACC"
  }
}
```

<ResponseField name="colors.primary" type="string" required>
  Primary brand color. Used for highlights, active sidebar items, and accents in light mode.
</ResponseField>

<ResponseField name="colors.light" type="string">
  Primary color used in dark mode.
</ResponseField>

<ResponseField name="colors.dark" type="string">
  Used for buttons and important interactive elements.
</ResponseField>

<ResponseField name="colors.background" type="object">
  Override the default page background color.
  <Expandable title="Background">
    <ResponseField name="light" type="string">Background color in light mode.</ResponseField>
    <ResponseField name="dark" type="string">Background color in dark mode.</ResponseField>
  </Expandable>
</ResponseField>

## Navbar

Configure top navigation bar links and the primary call-to-action button.

```json docs.json
{
  "navbar": {
    "links": [
      { "label": "Support", "href": "mailto:support@example.com" }
    ],
    "primary": {
      "type": "button",
      "label": "Dashboard",
      "href": "https://dashboard.example.com"
    }
  }
}
```

<ResponseField name="navbar.links" type="array">
  Text links shown in the top navigation bar.
  <Expandable title="Link">
    <ResponseField name="label" type="string">Display text.</ResponseField>
    <ResponseField name="href" type="string">Destination URL.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="navbar.primary" type="object">
  The prominent call-to-action in the top-right corner.
  <Expandable title="Primary CTA">
    <ResponseField name="type" type='"button" | "github"'>
      `button` renders a CTA button. `github` renders a GitHub star count badge.
    </ResponseField>
    <ResponseField name="label" type="string">Button label. Required when `type` is `button`.</ResponseField>
    <ResponseField name="href" type="string">URL the button or badge links to.</ResponseField>
  </Expandable>
</ResponseField>

## Footer

```json docs.json
{
  "footer": {
    "socials": {
      "x": "https://x.com/yourhandle",
      "github": "https://github.com/yourorg",
      "linkedin": "https://linkedin.com/company/yourco",
      "discord": "https://discord.gg/your-server"
    }
  }
}
```

<ResponseField name="footer.socials" type="object">
  Social icon links displayed in the footer. Supported keys: `x`, `github`, `linkedin`, `discord`, `slack`, `facebook`, `instagram`, `website`.
</ResponseField>

## Contextual toolbar

The contextual toolbar appears on every page and gives readers one-click access to AI tools. Configure it with the `contextual.options` field:

```json docs.json
{
  "contextual": {
    "options": ["copy", "view", "claude", "chatgpt", "mcp", "cursor", "vscode"]
  }
}
```

See [Contextual toolbar](/agent-ready/contextual) for the full list of available options.
