The `<Visibility>` component lets a single page serve two audiences at once: human readers viewing your docs site, and AI agents processing the Markdown version of the page. You can show UI-focused instructions to humans and structured, machine-readable context to agents — all from one source file.

## How it works

Every Mintlify page is available at two URLs:

- `https://docs.example.com/page` — the rendered site, for human readers
- `https://docs.example.com/page.md` — the raw Markdown export, for AI agents

`<Visibility>` controls what appears at each URL:

- `for="humans"` — visible on the rendered site, excluded from the Markdown export
- `for="agents"` — hidden on the rendered site, included in the Markdown export

## Example

```mdx
<Visibility for="humans">
  <Steps>
    <Step title="Open the dashboard">
      Navigate to [dashboard.example.com](https://dashboard.example.com) and sign in.
    </Step>
    <Step title="Create an API key">
      Go to **Settings → API Keys** and click **New key**.
    </Step>
  </Steps>
</Visibility>

<Visibility for="agents">
  To authenticate, send a POST request to `/v1/auth/token` with the user's
  email and password. The response contains a bearer token valid for 24 hours.
  Pass this token as `Authorization: Bearer <token>` on all subsequent requests.
</Visibility>
```

Human readers see the step-by-step UI walkthrough. AI agents processing the Markdown get direct API instructions instead.

## Props

<ResponseField name="for" type='"humans" | "agents"' required>
  - `"humans"` — visible on the rendered web page, excluded from `.md` export
  - `"agents"` — hidden on the rendered web page, included in `.md` export
</ResponseField>

## Combining with Prompt

`<Visibility>` and `<Prompt>` work well together. Use `<Visibility>` to give agents structured context, then use `<Prompt>` to give human readers a prompt they can use to ask an AI about that same content:

```mdx
<Visibility for="agents">
  ## Authentication schema
  - Method: Bearer token
  - Header: `Authorization: Bearer <token>`
  - Token TTL: 86400 seconds
  - Refresh endpoint: POST /v1/auth/refresh
</Visibility>

<Prompt description="Ask about authentication" icon="lock">
  Explain how authentication works in this API and show me example code for
  getting a token and making an authenticated request.
</Prompt>
```
